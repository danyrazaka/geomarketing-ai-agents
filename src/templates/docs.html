Documentation GeoMarketing AI Agents

Bienvenue dans la documentation de GeoMarketing AI Agents, une plateforme open source qui combine l'intelligence artificielle (DeepSeek R1) avec des outils d'analyse géospatiale pour optimiser vos décisions géographiques.

Cette documentation vous guidera à travers l'installation, l'utilisation et l'extension du projet.

Installation

Prérequis

  • Python 3.8 ou supérieur
  • pip (gestionnaire de paquets Python)
  • Git

Installation locale

# Cloner le dépôt
git clone https://github.com/username/geomarketing-ai-agents.git
cd geomarketing-ai-agents

# Créer et activer un environnement virtuel
# Sur Linux/macOS
python -m venv venv
source venv/bin/activate

# Sur Windows
python -m venv venv
venv\Scripts\activate

# Installer les dépendances
pip install -r requirements.txt

# Lancer l'application
python src/main.py

L'application sera accessible à l'adresse http://localhost:5000.

Analyse d'Emplacement Commercial

L'agent d'optimisation d'emplacement commercial vous aide à trouver le meilleur emplacement pour votre commerce en analysant diverses données géographiques et démographiques.

Comment utiliser l'analyse d'emplacement commercial

  1. Accédez à la page "Emplacement Commercial" depuis le menu principal
  2. Entrez la localisation que vous souhaitez analyser (ville, quartier, adresse)
  3. Sélectionnez le type de commerce (pharmacie, boulangerie, etc.)
  4. Ajustez les paramètres selon vos besoins (rayon d'analyse, facteurs d'importance)
  5. Cliquez sur "Analyser" pour lancer l'analyse
  6. Consultez les résultats sous forme de scores, cartes et recommandations
Facteurs d'importance
  • Population : Densité et caractéristiques démographiques
  • Concurrence : Présence de commerces similaires
  • Accessibilité : Facilité d'accès (routes, transports)
  • Visibilité : Exposition et visibilité du commerce
Résultats fournis
  • Scores d'attractivité : Global et par facteur
  • Carte interactive : Visualisation des emplacements optimaux
  • Carte de chaleur : Zones d'attractivité
  • Recommandations : Conseils stratégiques

Analyse de la Qualité des Sols

L'agent d'analyse de la qualité des sols vous aide à évaluer la compatibilité de vos terrains avec différentes cultures en analysant les propriétés pédologiques.

Comment utiliser l'analyse de la qualité des sols

  1. Accédez à la page "Qualité des Sols" depuis le menu principal
  2. Entrez la localisation de votre parcelle (ville, région, coordonnées)
  3. Sélectionnez le type de culture que vous souhaitez évaluer (stevia, blé, etc.)
  4. Ajustez les paramètres selon vos besoins (profondeur d'analyse, facteurs d'importance)
  5. Cliquez sur "Analyser" pour lancer l'analyse
  6. Consultez les résultats sous forme de scores, cartes de zones et recommandations agronomiques
Facteurs d'importance
  • pH : Acidité/alcalinité du sol
  • Drainage : Capacité d'évacuation de l'eau
  • Texture : Composition granulométrique
  • Matière organique : Richesse en nutriments
Résultats fournis
  • Scores de compatibilité : Global et par facteur
  • Carte interactive : Visualisation des zones optimales
  • Carte de qualité des sols : Zones de compatibilité
  • Recommandations agronomiques : Conseils par zone

Extension du Projet

Le projet est conçu pour être facilement extensible.

Intégration de nouvelles sources de données

Pour ajouter une nouvelle source de données, créez un nouveau module dans src/utils qui implémente les méthodes d'accès à cette source, puis intégrez-le dans les services existants.

Ajout de nouveaux types d'analyses

Pour ajouter un nouveau type d'analyse, créez un nouveau service dans src/services, une nouvelle route dans src/routes, et les templates correspondants dans src/templates.

Intégration avec d'autres modèles d'IA

Le client DeepSeek R1 est conçu pour être facilement remplaçable. Modifiez src/utils/deepseek_client.py pour intégrer un autre modèle d'IA, en conservant la même interface.

FAQ

Dans la version actuelle, les résultats sont basés sur des données simulées pour démonstration. Pour une utilisation en production, il est recommandé d'intégrer des sources de données réelles et de calibrer les modèles d'analyse.

Vous pouvez définir votre clé API DeepSeek R1 en tant que variable d'environnement DEEPSEEK_API_KEY, ou la fournir directement lors de l'initialisation du client dans src/utils/deepseek_client.py.

Oui, le projet peut fonctionner en mode hors ligne en utilisant des données simulées. Pour cela, assurez-vous que le paramètre use_mock est défini sur True dans les services.
